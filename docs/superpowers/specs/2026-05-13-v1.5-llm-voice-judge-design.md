**Date**: 2026-05-13 PM
**Status**: Approved design, ready cho plan
**Supersedes**: V1.3 body voice patch (partial — keeps V1.3 mechanical gates, drops prefer lists), V1.4 in-progress (keeps min_sentence_richness, drops V8 skill task)

## Context

User feedback 2026-05-13 (sau /tin STB + /tin FPT testing):

5 bài hôm nay xuất hiện 4 patterns lỗi:

### Pattern A — Bình dân giả (verb tự chế)
AI invent verbs from V1.2/V1.3 DRAMATIC_VERBS list mà không check ngữ cảnh:
- "chấm đích" (FPT title + body, MSN body) — không phải tiếng Việt tự nhiên
- "vọt lãi" (MSN title) — combo verb sai
- "xén lợi FPT mẹ" — kệch cỡn
- "VCBS chấm 111.421 đồng" — "chấm" = mark/dot, không phải định giá
- "đẻ ra tỉnh thứ tư" (FPT) — metaphor cưỡng ép
- "Playbook lặp từ Bình Định" — English word + verb lạ

### Pattern B+C — Hán-Việt formal + abbreviation không expand
- "độc bản" (di sản UNESCO) — Hán-Việt vs "duy nhất"
- "chưa hội đủ" — formal vs "chưa đủ"
- "tái định giá" — formal vs "định giá lại"
- "BCA" — Bộ Công An, không expand
- "GRDP" — Tổng sản phẩm địa bàn, không expand

### Pattern D — Fabricated data
- FPT bài: "tích lũy dưới 145 nghìn/cp" — FPT thực tế 70 nghìn. Master hallucinated price target.

### Pattern E — Length squeeze
V1.3 length cap -20% (flash 80-120 / qa 180-240 / listicle 220-280 / narrative 220-280) ép Master xé câu thành tail fragments → kết hợp với Pattern A tạo verb cưỡng ép.

### Root cause — user directive

> "Trong pipeline có nhiều chỗ bắt AI ép dùng list verb thì xem xét lại nhé, **định nghĩa thôi còn để AI linh động chứ**"

V1.2/V1.3 architecture over-prescriptive:
- `DRAMATIC_VERBS` + scorer "+2 dramatic_verb" → AI pile verbs from list regardless of fit
- `PREFERRED_BODY_VERBS` mapping table (V6.2) → force "ăn/khoe/dồn/xén" mọi chỗ
- `METAPHOR_MARKERS` scorer → force metaphor cưỡng ép

**Solution direction**: Shift from "prescription via word lists" to "definition via voice intent" + "LLM judge for nuanced semantic checks" + "mechanical for clear counts/regex".

## Approach picked

**Approach 3 — Pure LLM judge, Hybrid execution** (user picked Sonnet 4.6 + Hybrid)

- DROP word list enforcement (DRAMATIC_VERBS bonus, PREFERRED_BODY_VERBS, BAO_CHI_BODY_VERBS, BAO_CHI_FORMULAIC_PHRASES, CONCRETE_QUESTION_SUBJECTS, METAPHOR_MARKERS)
- KEEP mechanical gates (count/regex/structural): bold_density, em_dash_density, word_count, body_pattern, min_sentence_richness (V1.4), no_metadata_leak, actionable_closing, has_orphan_number
- ADD LLM judge (Sonnet 4.6) for voice quality: `judge_body_voice`, `judge_title_voice`
- ADD 2 new gates: `check_abbreviation_expanded`, `check_price_realistic`
- REVERT length cap to V5.0

**Cost**: ~$0.02/article × 2-3 articles/tin = ~$0.06/run, ~$0.24/tin-hot N=4. Acceptable.

## Architecture

```
Master agent → body + title
            ↓
check_all_v5(body, title, format_id, stance) — V1.5
            ↓

┌── MECHANICAL (regex/count/structural, fast, deterministic):
│   ├── bold_density               (count `**X**` per format target)
│   ├── em_dash_density            (count `—` ≤ 1/100 words)
│   ├── word_count                 (per format V5.0 ranges)
│   ├── body_pattern               (structural: opening + bullets + closing)
│   ├── min_sentence_richness      (V1.4: <20% sentences <10 từ)
│   ├── no_metadata_leak           (specific enum tags ban)
│   ├── actionable_closing         (stance verb + quantified trigger + no vague)
│   └── has_orphan_number          (V1.3: số/percent must have subject)
│
├── LLM JUDGE (Sonnet 4.6, ~$0.005-0.02/call, semantic):
│   ├── judge_body_voice           (replaces bao_chi_body — flags fabricated verbs)
│   ├── judge_title_voice          (replaces is_bao_chi + hook_strong + binh_dan nested)
│   └── check_no_hedging           (V5.1.2 — already LLM)
│
└── NEW gates V1.5:
    ├── check_abbreviation_expanded (regex find 3-4 upper, check expansion or allowlist)
    └── check_price_realistic       (Finpath API current price ± 50%)
```

## Voice intent definition (master prompt)

Replace V1.4 prescriptive rules with definition-only:

```
## Voice — bình dân xuồng xã nguy hiểm (V1.5 definition)

Reader is everyday Vietnamese investor (NĐT). Body must read naturally —
NOT thông cáo báo chí, NOT AI-generated. Apply intent:

- USE: concrete Vietnamese verbs everyday people use ("ăn lãi/cắt giảm/
  bán cổ phiếu/giảm nợ"). Pick verbs THAT FIT the action, not from a list.

- AVOID: invented verb-noun combos that don't exist in natural Vietnamese.
  Bad examples (V1.5 audit): chấm đích / vọt lãi / xén lợi / đẻ ra tỉnh /
  Playbook lặp từ / VCBS chấm 111.421 đồng. If reader chưa từng nghe câu
  đó trên báo/đời sống, rewrite.

- AVOID Hán-Việt formal: độc bản → duy nhất / hội đủ → đủ /
  tái định giá → định giá lại / cấu trúc → cơ cấu / hội đủ điều kiện →
  đủ điều kiện / phương án xử lý → cách xử lý / triển khai → làm /
  ban hành → ra / thông qua nghị quyết → quyết. Replace với word everyday
  Vietnamese.

- AVOID báo chí buzzwords + thông cáo verbs: bàn giao / ghi nhận /
  công bố / dự kiến đạt / phát hành thành công / lao dốc / bứt phá /
  lập kỷ lục / phấn đấu.

- EXPAND abbreviations first occurrence: "Bộ Công An (BCA)" lần đầu,
  sau đó BCA. Same với GRDP / SCIC / EBITDA / SOE / etc. Allowlist
  (không cần expand): ESOP / NIM / ROE / ROA / EPS / IPO / CASA / NPL.

- VERIFY current price from Finpath API trước khi quote price target
  trong closing. Target phải trong ±50% current. NEVER fabricate price.

- METAPHOR optional, not mandatory. Use khi natural ("gấp 3 lần
  Vietcombank cùng kỳ", "ngang ngửa BIDV"), drop khi cưỡng ép
  ("đẻ ra tỉnh thứ tư" = forced).
```

## LLM judge prompt design

### judge_body_voice (called per body)

```
You are a Vietnamese finance editor. Read this article body and judge voice quality.

Body: {body}
Format: {format_id}
Stance: {stance}

Check 4 dimensions:
1. Natural Vietnamese verbs (NOT invented combos like "chấm đích / vọt lãi / xén lợi")
2. NO Hán-Việt formal vocabulary (độc bản / hội đủ / tái định giá / etc.)
3. NO báo chí thông cáo style (bàn giao / ghi nhận / công bố pile-on)
4. Voice intent "bình dân xuồng xã + sharp" — sounds like senior analyst
   talking to ordinary investor, NOT a press release

Return JSON:
{
  "pass": true/false,
  "issues": [array of specific quotes that failed],
  "reasoning": "1-2 sentence Vietnamese explanation"
}
```

### judge_title_voice (called per title)

```
You are a Vietnamese finance editor. Score this hook quality 1-10.

Title: {title}
Body excerpt (first 200 chars): {body_excerpt}

Check:
- Ticker present (REQUIRED)
- ≤16 từ
- No orphan number/percent (number must have subject)
- No invented verb combos
- No báo chí formulaic phrases (đánh đổi gì / hy sinh để / lao dốc)
- Sharp tension / curiosity hook
- Bình dân — everyday Vietnamese, not formal

Return JSON:
{
  "pass": true/false (passes hard criteria),
  "score": 1-10,
  "issues": [array of specific quotes],
  "reasoning": "1-2 sentence Vietnamese"
}
```
